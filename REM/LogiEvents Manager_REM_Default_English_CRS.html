UC-0001 Agregar Evento
Version 1.0 ( 24/03/2025 )
Authors
  • Sergio Andrés Zúñiga Castillo
Sources
  • LogiEvents
Dependencies None
Description The system shall behave as described in the following use case when un administrador quiere agregar un evento a la lista de eventos disponibles
Precondition Permisos de administrador
Log in con cuenta válida
Ordinary sequence Step Action
1 Actor Administrador (ACT-0001) Ingresa a la pantalla para agregar evento
2 The system Despliega el formulario con los datos para registrar un nuevo evento (Nombre, Fecha, Hora, Descripción, Ubicación, Capacidad, Categoría, Estado, Precio, Imagen/Banner)
3 Actor Administrador (ACT-0001) Ingresa los datos del evento en el formulario y presiona el botón de confirmar
4 The system intenta agregar el nuevo evento al sistema
5 The system al insertar correctamente el evento, notifica al administrador y actualiza las listas en la página.
Postcondition Agregar el evento a la base de datos
Exceptions Step Action
2 If hay un error en el despliegue del formulario, the system recargar la página, then this use case resumes
4 If hubo un error al insertar el evento, the system despliega un mensaje de error y solicita intentar de nuevo, then this use case is cancelled
Performance Step Maximum time
- -
Expected frequency 5 times per week(s)
Importance vital
Urgency under pressure
Status under construction
Stability high
Comments None

UC-0002 Reservar Evento
Version 1.0 ( 24/03/2025 )
Authors
  • José Ignacio Madriz Ortiz
Sources
  • LogiEvents
Dependencies None
Description The system shall behave as described in the following use case when un usuario se quiere inscribir a participar en un evento
Precondition Tener una cuenta de LogiEvents
Cupos disponibles en el evento
Log in con cuenta válida
Ordinary sequence Step Action
1 Actor Usuario (ACT-0002) selecciona un evento de la página principal de eventos
2 The system enseña las propiedades del evento junto a la opción para reservar evento
3 Actor Usuario (ACT-0002) selecciona la opción de reservar evento
4 The system pide al usuario confirmar su correo, nombre y teléfono e indicar la cantidad de espacios en la reserva
5 Actor Usuario (ACT-0002) llena el formulario pedido
6 The system pide al usuario un número de teléfono para enviar un código de verificación
7 Actor Usuario (ACT-0002) ingresa el número de teléfono, selecciona enviar y espera el mensaje para poder insertar el código de confirmación
8 If si la palabra de confirmación es correcta, the system realiza la transacción para agregar la reserva y envía la confirmación al correo del usuario
Postcondition Actualizar y agregar la reserva a la base de datos.
Exceptions Step Action
2 If no se encuentra el evento, the system pedir al usuario que vuelva a consultar en unos minutos, then this use case is cancelled
5 If formato incorrecto de correo electrónico o teléfono, the system pide reingresar datos válidos, then this use case is cancelled
7 If palabra de verificación incorrecto, the system sugerir enviar el mensaje de nuevo o reintetar
, then this use case is cancelled
Performance Step Maximum time
- -
Expected frequency 20 times per day(s)
Importance vital
Urgency under pressure
Status under construction
Stability high
Comments None

, then this use case is cancelled
4 If se encuentra un error al actualizar la base de datos, the system notifica al usuario que hubo un error y pide volver a insertar la información
, then this use case is cancelled

UC-0004 Registrar Usuario
Version 1.0 ( 24/03/2025 )
Authors
  • Fernando Andrés Gonzales Robles
Sources
  • LogiEvents
Dependencies None
Description The system shall behave as described in the following use case when un usuario nuevo quiere ingresar al sistema
Precondition Tener nombre de usuario, documento de identidad, contraseña, correo electrónico y número de teléfono
Que no exista un usuario con mismo documento de identidad o correo registrado
Ordinary sequence Step Action
1 Actor Usuario (ACT-0002) dentro del log in, ingresa a la opción de registrar usuario
2 The system muestra un formulario que pide la información del usuario (Nombre, Nombre de usuario, identificación, contraseña, correo electrónico, y número de teléfono)
3 Actor Usuario (ACT-0002) llena el formulario con sus datos y presiona registrar
4 The system verifica la información en el sistema, que no se repitan ciertos campos como correo electrónico, identificación, nombre de usuario y teléfono
5 The system guarda la información en la base de datos y notifica al usuario que se guardó exitosamente
Postcondition Agregar el nuevo usuario a la base de datos
Exceptions Step Action
4 If si hay campos repetidos o se encuentra algún error por parte de la base de datos, the system indica al usuario que lo intente de nuevo, en caso de información repetida, indicar cuál está repetida, then this use case is cancelled
5 If hay un error en la actualización de la base de datos, the system notificar al usuario y pedir que lo intente de nuevo
, then this use case is cancelled
Performance Step Maximum time
- -
Expected frequency 15 times per day(s)
Importance vital
Urgency immediately
Status under construction
Stability high
Comments None
